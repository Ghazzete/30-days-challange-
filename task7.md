5 Core Concepts (Short Answers)
1. /constitution

Is hissa mein project ka base define hota hai. Yani project ka purpose kya hai, kis problem ko solve karega, aur iska main goal kya hai. Constitution basically poore project ki foundation hoti hai.

2. /specify

Is stage me hum detail me likhtay hain ke system ko exactly kya karna chahiye. Yani requirements, rules, expected behaviorâ€”sab kuch clear taur par define kiya jata hai. Isse project me ambiguity nahi rehti.

3. /plan

Yahan planning hoti hai ke project ko kaise build kiya jayega. Kaun se steps follow honge, kis cheez ko kitni priority milegi, aur workflow kaisa hoga. Planning se development smooth ho jati hai.

4. /tasks

Plan ko chhoti chhoti tasks me break kiya jata hai. Har task simple aur action-based hota hai, jisse work manageable ban jata hai. Yeh part team ko clear direction deta hai ke kis ne kya karna hai.

5. /implement

Is stage me actual development hoti hai. Yani code likhna, features banana, aur project ko functional form me convert karna. Jo kuch upar define aur plan hua hota hai, usay yahan practically build kiya jata hai.
